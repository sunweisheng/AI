# OpenClaw在私聊（DM）和群聊（Group）中加载文件和记忆机制

## 核心机制对比表
| 机制/文件 | 私聊 (DM) / 跨渠道 | 群组 (Group Chat) |
| :--- | :--- | :--- |
| Session 类型 | 共享的 Main Session<br>(所有配对用户的私聊折叠至此) | 独立的隔离 Session<br>(每个群组默认使用独立的 Session) |
| MEMORY.md (长期记忆) | ✅ 加载<br>(持久化存储，记录决策与偏好) | ❌ 不加载<br>(防止在公共场合泄露私人长期记忆) |
| USER.md (用户信息) | ✅ 加载<br>(读取称呼、偏好、关系) | ❌ 不加载<br>(避免将个人偏好带入公共对话) |
| Daily Logs (日常日志) | ✅ 加载<br>(读取今日+昨日日志，保持连续性) | ✅ 加载<br>(读取今日+昨日日志，保持上下文) |
| SOUL.md / AGENTS.md | ✅ 加载<br>(定义核心人格与边界) | ✅ 加载<br>(定义核心人格与边界) |
| Skills (技能) | ✅ 扫描加载<br>(工作区 > 用户级 > 内置) | ✅ 扫描加载<br>(同上) |

## 详细加载逻辑解析
1. 私聊场景
* 记忆继承：私聊被设计为你和 Agent 的“私密空间”。系统会将所有已配对用户的私聊折叠到一个共享的 main session 中。
* 跨渠道一致性：同一个用户在 Telegram 和 WhatsApp 的私聊会共享这个 main session，这意味着你在不同平台上的记忆和偏好是同步的。
* 文件行为：
  * MEMORY.md 会被加载，Agent 会记得你在私聊中积累的长期事实、决策和偏好。
  * USER.md 会被加载，Agent 会根据你的结构化信息（如称呼、关系）来调整回复。
2. 群聊场景
* 隔离机制：每个群组默认使用独立的隔离 Session。
* 隐私保护：
  * MEMORY.md 不会被加载。这是为了确保 Agent 在群聊中不会意外泄露你在私聊中告诉它的私人信息或长期记忆。
  * USER.md 不会被加载，因为群聊是公共场合，不应应用针对个人的特定偏好。
* 响应规则：
  * 默认情况下，Agent 在群聊中只响应 @Agent名称 的消息，忽略其他闲聊内容。
  * 可以通过指令 /activation mention|always 切换为“响应所有消息”模式（但这会消耗更多 Token）。
3. 通用加载项 (私聊与群聊均加载)
* Daily Logs：无论是私聊还是群聊，系统都会读取“今日”和“昨日”的日志文件，以确保对话的短期连续性。
* Skills：系统会扫描并加载技能。优先级逻辑为：工作区级 Skills > 用户级 Skills > 内置 Skills。如果同名技能存在于多个层级，高优先级会覆盖低优先级。
* SOUL.md & AGENTS.md：定义 Agent 核心人格、边界和回复风格的文件，在任何会话开始时都会加载。
