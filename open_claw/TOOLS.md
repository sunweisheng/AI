# TOOLS文件

TOOLS是AGENTS文件的补充，它记录的不是行为规则，而是这台机器上具体的环境备忘录，文件路径、工具名称、账号用途。  
在工作中，遇到Open Claw不知道具体路径或工具名称时陆续补充即可。

## TOOLS默认模版
````markdown
# TOOLS.md - 本地笔记

技能定义工具**如何**工作。这个文件是记录**你的**专属内容 — 那些只属于你的设置信息。

## 这里放什么

比如：
- 摄像头名称和位置
- SSH 主机和别名
- TTS 语音偏好
- 音箱/房间名称
- 设备昵称
- 任何环境特定的内容

## 示例
markdown
摄像头
客厅 → 主要区域，180°广角
前门 → 入口，运动触发
SSH
家庭服务器 → 192.168.1.100，用户名：admin
TTS
首选语音："Nova"（温暖，略带英式口音）
默认音箱：厨房 HomePod
## 为什么要分开？
技能是共享的。你的设置是你的。把它们分开意味着你可以更新技能而不会丢失你的笔记，分享技能时也不会泄露你的基础设施信息。
---
添加任何有助于你工作的内容。这是你的备忘单。
````

## 注意事项
不要放任何密码、API KEY或其他敏感信息，这些内容应该保存在.env等专门存储密钥的文件中。  
TOOLS只是记录，工具在哪里，如何调用，好几次Open Claw都把强制执行的规则写在TOOLS里，这是错误的，应写在AGENTS里。
