# MEMORY
OpenClaw的记忆不是自身具备的能力，它依赖一套需求主动搭建和维护的文件系统。  
* MEMORY.md用于存放核心信息，控制在200行以内，每次对话时自动加载。
* memory/YYYY-MM-DD.md用于记录每天的具体事项。
* AGENTS.md是整个系统的驱动力——没有明确的规范，OpenClaw不会主动记录。
* 对于大多数“OpenClaw忘事了”的问题，其根源是信息没有被记录。
记忆系统的质量取决于两件事：你在AGENTS中建立的记录规范，以及你投入的维护时间。规范约清晰，维护越频繁，OpenClaw的表现就越稳定。

## 两层记忆
第一层：日记（memory/YYYY-MM-DD.md）。OpenClaw每天都会生成一个文件，记录当天发送的具体事情，这是被动记忆。
第二层：长期记忆（MEMORY.md）OpenClaw从日记中提炼核心信息，例如你的偏好习惯、重要决策原因、踩过的坑，并将它们存储在长期记忆文件中，这个文件在每次会话时都会自动加载，所以必须精简（控制在200行左右），这是主动记忆。

## 建议的记忆规范
在OpenClaw中输入以下内容，让OpenClaw自己更新AGENTS.md：
````markdown
你在每次会话开始时都会重新启动。以下文件是你的记忆延续：
* 每日笔记： `memory/YYYY-MM-DD.md`（需要时创建 `memory/` 目录）— 记录当天发生的事情的原始日志。写下来——别只靠“脑子里记”。
* 记忆有限——想记住某事，就要写到文件里。
* “脑子里记”无法在会话重启之后保留，文件可以。
* 当有人说“记住这个”就好更新memory/YYYY-MM-DD.md或对应的文件。
* 当你学到经验就要更新AGENTS.md、TOOLS.md或相关技能文档。
* 当你犯错就记录下来，避免未来的你重蹈覆辙。

- **每日笔记：** `memory/YYYY-MM-DD.md`（需要时创建 `memory/` 目录）— 发生的事情原始记录
- **长期记忆：** `MEMORY.md` — 你的整理记忆，像人类的长期记忆记录重要的事情。决策、上下文、需要记住的事情。除非被要求保密，否则跳过秘密。
**文字 > 大脑** 📝
````

## 注意事项
* OpenClaw没有自动写日记的原因一般是AGENTS中没有明确的要求导致。
* 在记忆文件中找不到想要的内容，大概率是没有记录到文件中。
* 直接编辑记忆文件是可以的，但一般让OpenClaw自己更新记忆文件。
* 不要将敏感信息存在记忆文件中。
