````shell

openclaw config set channels.feishu.footer.status true

# 添加员工飞书机器人配置

# 飞书机器人staff-b配置
openclaw config set channels.feishu.accounts.staff-b.appId "cli_a95917690c7xxxxx"
openclaw config set channels.feishu.accounts.staff-b.appSecret "SdC9aBZRn5GxppfRd7JMYbvW1NJxxxxx"

# 创建独立有人格的员工Agent
# 创建名为staff-a-bot的Agent并绑定飞书机器人staff-a
openclaw agents add staff-a-bot --bind feishu:staff-a --workspace ~/.openclaw/staff-a-bot-workspace
# 创建名为staff-b-bot的Agent并绑定飞书机器人staff-b

# 重启网关
openclaw gateway restart
````

### 修改配置文件内容
配置文件位置:~/.openclaw/openclaw.json  
需要修改的配置节点: "tools"，添加内容：
````json
{
    "tools": {
    "sessions": {
      "visibility": "all" 
    },
    "agentToAgent": {
      "enabled": true,
      "allow": ["main", "staff-a-bot", "staff-b-bot"]
    }
  }
}
````
visibility:Agent之间的可见性，主Agent（main）可以看到所有的Agent。  
allow：可以通讯的Agent列表，他们都可以与主Agent（main）通讯。

需要修改的配置节点："session"，修改内容：
````json
{
  "session": {
    "dmScope": "per-account-channel-peer"
  }
}
````
per-account-channel-peer:私聊信息和记忆在不同沟通渠道的不同机器人之间隔离，默认是私聊信息在不同沟通渠道之间隔离但同一个渠道中的不同机器人之间会叠加共享信息和记忆（per-channel-peer），简单说就是私聊的信息和记忆将在“不同渠道”以及“同一渠道的不同机器人”之间实现完全隔离。  
上述修改完成之后重启网关，通过飞书与AI经理、员工A、员工B分别进行对话，初始化人格角色信息。
